# MVP PRD v1

## Product positioning
A lightweight H5 operational-control tool for restaurant owners managing 2–5 stores.

## One-sentence product definition
Help bosses see the biggest weekly issue, assign the next best action, and review execution without reading complex dashboards.

## Core loop
Manager enters data → system identifies issues → boss sees action cards → boss assigns → manager feeds back → weekly review.

## Target users
### Boss / investor
- manages 2–5 stores
- still deeply involved in operations
- does not want complex BI
- wants fast decisions and execution visibility

### Store manager
- responsible for execution
- can input data but wants low-friction workflows
- needs clear daily/weekly todo items

## MVP must-do
- boss overview
- store comparison
- store detail
- action cards
- task assignment
- manager data entry
- OCR-assisted entry
- task feedback
- weekly review

## MVP explicitly not-doing
- automated full reconciliation
- procurement forecasting
- inventory engine
- HR/attendance
- dynamic real-time gross margin
- heavy HQ systems

## Success criteria
### For restaurant owners
- can identify the key store issue quickly
- can assign 1–3 actions weekly
- can see whether actions were executed
- can review weekly results

### For investors / internal validation
- real stores use the flow
- at least one full weekly loop is completed
- action cards lead to actual manager tasks
- product shows standardizable potential
